# Игра по сети с друзьями

## Самый простой способ — Render (рекомендуется)

Игра задеплоена на Render, поднимать у себя ничего не нужно. Просто открой
ссылку и отправь её друзьям:

**https://ancient-greece-boardgame.onrender.com**

1. Открой ссылку → «Создать игру» → выбери число игроков → введи имя. Попадёшь
   в **лобби**.
2. Нажми **«Скопировать ссылку»** и отправь друзьям. Каждый открывает её, вводит
   имя и занимает место.
3. Когда все за столом, **хост** жмёт **«Начать игру»** — заставка, очерёдность,
   партия пошла.

Нюансы бесплатного плана Render:

- **Засыпает после ~15 минут простоя.** Первый заход после сна грузится
  ~30–60 секунд (сервер просыпается), дальше всё быстро. Пока вы играете — он не
  спит.
- **Партии хранятся в памяти.** При засыпании/перезапуске сервера активные матчи
  теряются — на одну игровую сессию этого достаточно, «продолжить завтра» уже не
  выйдет.
- Деплой обновляется автоматически при `git push` в `master`.

Дальше — **запасной способ** на случай, если не хочется ждать пробуждение Render
или нужна разработка с локальным сервером.

## 1. Запустить игру у себя

```
npm install        # один раз
npm run play       # собирает клиент и поднимает сервер на http://localhost:3001
```

Проверь, что открывается `http://localhost:3001` — это и есть игра.

> `play` отдаёт собранную версию. После правок в коде — снова `npm run play`.
> Если клиент уже собран, можно просто `npm run serve` (без пересборки).

## 2. Открыть наружу — ngrok (проверено, рабочий способ)

На этой сети провайдер режет порт 7844, поэтому Cloudflare Tunnel не
поднимается, а localtunnel нестабилен. **ngrok** работает: он идёт одним
TLS-соединением через 443.

ngrok уже установлен (`winget install Ngrok.Ngrok`) и токен сохранён в конфиг
(`ngrok config add-authtoken …`). Закреплён **постоянный домен**, запускай так:

```
ngrok http 3001 --url=operate-backyard-equate.ngrok-free.dev
```

Постоянная ссылка на игру — всегда одна и та же:
**https://operate-backyard-equate.ngrok-free.dev**

> Без `--url` ngrok выдаст случайный поддомен; с `--url` адрес фиксирован.

> Должны работать **оба** окна одновременно: сервер (`npm run serve`/`play`) и
> `ngrok http 3001`. Компьютер — включён, пока идёт партия.

## 3. Играть

1. Открой ngrok-ссылку сам. На бесплатном тарифе ngrok один раз покажет
   страницу-предупреждение — нажми **«Visit Site»**.
2. «Создать игру» → выбери число игроков → введи своё имя. Попадёшь в **лобби**.
3. Нажми **«Скопировать ссылку»** и отправь друзьям. Каждый у себя: открывает →
   **«Visit Site»** (один раз) → вводит имя → занимает место.
4. Когда все за столом, **хост** жмёт **«Начать игру»**. Проиграется заставка,
   определится очерёдность хода — и партия начинается.

Состояние партии хранится на сервере: если вкладка закрылась — вернись по той
же ссылке и продолжишь.

## Нюансы

- **Ссылка постоянная** — закреплён статический домен
  `operate-backyard-equate.ngrok-free.dev`, он используется при запуске с
  флагом `--url=…` (см. выше). Без флага ngrok выдаст случайный адрес.
- **Свой ngrok-адрес со своего же ПК может не открыться** (NAT hairpin) — это
  нормально, друзей снаружи не касается. Проверяй с телефона по мобильному
  интернету или попроси друга.
- **Туман войны** прячется на стороне клиента. Для друзей этого достаточно, но
  строго скрытым он не является — при желании вынесем фильтрацию на сервер
  (`playerView`) отдельной задачей.
- **Хотсит** (все за одним компом) доступен всегда: `http://localhost:3001/#/local`.

## Запасные варианты туннеля

- **Локальная сеть (без интернета):** если все в одном Wi-Fi — друзьям хватит
  `http://ТВОЙ_LAN_IP:3001` (узнать IP — `ipconfig`), разреши порт в брандмауэре.
- **Cloudflare Tunnel** (`cloudflared tunnel --url http://localhost:3001`) —
  бесплатный и без регистрации, но на текущей сети не проходит (порт 7844).
  Сработает с другой сети / мобильной раздачи.

## Если друг не подключается

- Убедись, что запущены оба окна (сервер и ngrok) и ПК не ушёл в сон.
- Друг должен нажать «Visit Site» на странице ngrok.
- Глянь логи ngrok в его окне (видно входящие запросы и ошибки).
